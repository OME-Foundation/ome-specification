# Organisational Memory Engineering

This repository is the public source of truth for research and future public
specifications concerning Organisational Memory Engineering (OME).

## The problem

Organisations depend on reasoning that is distributed across people,
conversations, decisions, requirements, documents, systems and routines. As
work crosses teams and changes over time, the connections between an objective,
the evidence considered, the decision made, its implementation and its outcome
are often weakened or lost.

The consequences include repeated discovery, decisions that cannot be
explained or challenged, requirements detached from their original purpose,
obsolete knowledge applied as if it were current, and lessons that fail to
change later work.

Existing fields address important parts of this problem: organisational memory
and learning, knowledge management, requirements traceability, decision
rationale, provenance, cognitive systems engineering, enterprise architecture
and semantic modelling. The open question is whether these can be integrated
into a useful engineering practice for preserving, retrieving, testing and
revising organisational reasoning across a delivery lifecycle.

OME is presently an engineering synthesis and research programme investigating
that question. It is not claimed as a proven new discipline.

## Repository boundary

This public repository contains the OME framework, terminology, principles,
models, metrics, specifications, research, and public contribution material.

Product-specific strategy, commercial plans, implementation architecture,
security details, delivery backlogs, and source code for private reference
implementations belong in their respective private repositories.
