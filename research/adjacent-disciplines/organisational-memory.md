# Organisational memory

## Established definition and frameworks

### Source findings

Walsh and Ungson define organisational memory through processes of information
acquisition, retention, and retrieval. Their influential model locates retained
information in individuals, culture, transformations, structures, ecology, and
external archives. They also warn that memory can be used, misused, or abused;
retention is not automatically beneficial.

This literature makes several points already established before OME:

- organisations retain information beyond any one employee;
- retention is distributed across people, routines, structures, culture, and
  records rather than residing in a single repository;
- retrieval and use matter as much as storage;
- memory is selective, political, and capable of preserving error;
- forgetting can be adaptive as well as harmful.

### OME interpretation

OME must adopt **organisational memory**, **acquisition**, **retention**,
**retrieval**, **retention facilities**, and **organisational forgetting**
rather than rename them. A software platform cannot itself be "the"
organisational memory; at most it supports selected memory processes and
retention facilities.

The founding emphasis on preserving reasoning adds a useful operational focus,
but is not a replacement definition for organisational memory. OME may focus on
the engineered qualities of memory across delivery work: provenance,
retrievability, validity, change, and consequence.

## Problems handled well

- Explaining how past information influences present decisions.
- Showing why repositories alone do not create usable memory.
- Identifying social and structural locations of retained knowledge.
- Explaining both beneficial memory and competency traps or stale routines.

## Limitations relevant to OME

The classic model is conceptual rather than an implementable measurement
standard. It gives limited guidance for end-to-end digital traceability,
machine-assisted reconstruction, evidence quality, or continuous validation of
memory in modern socio-technical delivery systems.

## OME overlap and possible gap

| Founding idea | Assessment |
| --- | --- |
| Organisational memory can be engineered | **Hypothesis**: plausible but requires intervention and outcome evidence. |
| Memory is more than documentation | **Established**. |
| Preserve reasoning, not only artefacts | **Principle**, supported by decision-rationale literature. |
| A canonical organisational-memory lifecycle | **Synthesis** if mapped to acquisition, retention, retrieval, use, updating, and forgetting. |
| A measurable memory capability | **Open question** pending validated constructs. |
